# ESPN Workflow Run Policy

## Scope
- Workflow: `.github/workflows/update_espn_cbb.yml`
- Purpose: build ESPN CBB core data, derived feature tables, and optional rankings with explicit artifact contracts.

## Trigger Policy
- `schedule` (nightly): runs data update and rankings, uploads artifacts, **does not commit data**.
- `workflow_dispatch` (manual): runs update and optional rankings; can commit curated CSV outputs only when `commit_data=true`.

## Why Data Commits Are Restricted
- Scheduled runs can produce frequent data churn and repository growth.
- Defaulting to artifact-only publishing protects `main` from runaway commits.
- Manual commits are constrained by:
  - curated allowlist (stable CSV outputs only),
  - max staged payload guard (`50 MB`),
  - safe push flow (`git pull --rebase` once, fail on conflicts).

## Dependency Graph (Artifact Contracts)
- `espn_pipeline`
  - produces `data/games.csv`, `data/team_game_weighted.csv`, `data/team_pretournament_snapshot.csv`
- `rotation_features`
  - requires player-level logs/metrics
  - produces `data/rotation_features.csv`
- `situational_features`
  - requires `data/team_game_weighted.csv`
  - produces `data/situational_features.csv`
- `luck_regression_features`
  - requires `data/team_game_weighted.csv`
  - produces `data/luck_regression_features.csv`
- `player_availability_features`
  - requires `data/player_game_logs.csv`
  - produces `data/player_availability_features.csv`
- `season_summaries`
  - requires `data/team_game_weighted.csv`
  - produces `data/player_season_summary.csv`, `data/team_season_summary.csv`
- `rankings`
  - requires fresh `data/team_game_weighted.csv` and `data/team_pretournament_snapshot.csv`
  - produces `data/cbb_rankings.csv`, `data/cbb_rankings_by_conference.csv`

## Execution Model
- Stage order is derived from `.github/pipeline_contracts/espn_pipeline_contract.json`.
- `.github/scripts/run_pipeline_contract.py` enforces:
  - required input existence,
  - minimum row thresholds,
  - required columns,
  - freshness windows where configured.
- Each job emits a manifest (`data/update_manifest.json`, `data/rankings_manifest.json`) plus integrity summaries.

## Health Checks
- Required artifact files are validated with `.github/scripts/check_csv.sh`.
- Compact integrity block is generated by `.github/scripts/print_data_integrity.py`.
- Workflow lint/validation runs via `.github/scripts/validate_workflows.py` when present.

## Known Constraints
- `days_back > 180` is allowed only for manual dispatch backfills.
- Rankings job will not run if update artifact or required inputs are missing/stale.
- Contract runner currently executes shell commands directly; script-level nondeterminism is outside workflow control.
